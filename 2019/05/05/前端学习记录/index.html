WebStorm局域网手机真机调试

1.打开webstorm–>Preferences–>Build,Execution,Deployment–>Debugger
2.在Built-in server中配置Port为没有占用的端口号，勾选里面的俩个选项(Can accept external connections; Allow unsigned requests)
3.在Deployment照片你高点击+号，设置type为Local or mounted folder
a.name:自定义；
b.type:Local or mounted folder；
c.Upload/download project files: Folder:你项目所在父级路径(注意：是父级路径)；
d.Browse files on server:Web server root URL:http://ip地址:debugger中设置的port端口号，这个域名是局域网，也是隶属于同一个路由发射出来的网络所分配的ip地址；
4.匹配路径设置：点击Mappings: Local path:你项目所在父级路径(注意：是父级路径)；
5.设置完成，点击apply，ok，在项目中使用浏览器打开即可，手机输入地址即可产看

-moz 、-ms、-webkit什么意思

  • 1、-moz代表firefox浏览器私有属性
  • 2、-ms代表ie浏览器私有属性
  • 3、-webkit代表safari、chrome私有属性
  • 这些是为了兼容老版本的写法，比较新版本的浏览器都支持直接写：border-radius。
  • 现在比较流行的一个概念就是：不太关键的样式，可以不考虑兼容，比如你说的圆角，并不影响内容的显示，如果客户浏览器太旧，就让他看到旧的方形效果就是，一定要看到新效果，只能建议客户升级浏览器。

绝对定位(子绝父相)

一般使用子绝父相的定位方式,就是子类假如使用绝对定位,父类一般使用相对定位,否则子类会向外层一直找,直到找到相对定位的控件,否则会以windon作为参照来定位.

Component is not found in path

问题：Component is not found in path “pages/circle/XXX” (using by “pages/circle/circle”);onAppRoute Error: Component is not found in path “pages/circle/XXX”
1.检查.json文件中usingComponents的配置是否正确，
2.一般出现问题是配置错误，导致找不到错误路径的自定义组件
“usingComponents”: {
“component-tag-name”: “path/to/the/custom/component”
}

超出屏幕一定距离

问题：width:100%,这时候设置margin会导致超出屏幕margin的距离
方案：父布局设置relative，子布局设置absolute，设置left/right

PC适配注意事项(使用chrome调试后再用Firefox检查)

1.字体在Firefox上最小是1px，chrome上最小是12px
2.有文字的区域不要使用固定高度
3.在固定高度的情况下尽量不要使用固定宽度，或者固定padding-left/margin-left
4.多元素列表使用flex-warp，item及内子元素(若有元素的宽占比过大的情况下，如图片占比超过一半屏宽)
5.position:fixed的情况下注意向下或者向右超出

前端适配手机

1.使用@media only screen and(max-width: 1024px) and(min-width:959px){}实现页面自适应布局时：1024px既适配于**PC也适配于手机**
建议：PC和手机写两套来适应。
2.使用rem时animation：transform移动距离很小，或者修改不生效
解决：可能-缓存问题，删除缓存
3.iPhone加载animation时css会在初始化就配置(产生的位移很小)
解决：使用addClass，setTimeout延时加载==等页面加载出来后在设置动画
4.chrome浏览器的响应式设计模式与真机不一样—>使用Firefox即可
iPhone可以使用Safari的开发模式：Safari的开发模式-偏好-高级-底部勾选开发，工具栏-开发-进入响应式设计模式
